
\chapter{Étude des besoins}


\section{Introduction}

L'étape de l'analyse des besoins est l'une des étapes les plus importantes à considérer.

\smallskip

En effet, si les besoins sont mal spécifiés et exprimés, ou mal analysés, toute la suite sera mal traitée, d'où l'importance accordée à cette activité. 

\smallskip

Dans cette partie notre objectif est de présenter les fonctionnalités estimé du futur application à développer. 

\bigskip


\section{Spécification des besoins}

Pour assurer la réussite de notre projet, il est essentiel que nous parvenions à une vue claire des différentes besoins escomptés de notre application.

\medskip
\medskip

Nous présentons dans ce qui suit tout les besoins fonctionnels et non fonctionnels.


\bigskip

\subsection{Besoins fonctionnels}

L'application à développer doit assurer les fonctionnalités suivantes : 

\medskip
\medskip

\begin{itemize}
\item \textbf{ Gérer les missions :}
\medskip
\medskip
\begin{itemize} \item Consultation des missions \item Ajout des nouveaux missions \item Modification des missions existantes \item Supprimer des missions existantes \item Rechercher des missions par Code, Libellé ou Type  \end{itemize}
\medskip
\item \textbf{ Gérer les partenaires : }
\medskip
\medskip
\begin{itemize} \item Consultation des partenaires \item Ajout des nouveaux partenaires \item Modification des partenaires existantes \item Supprimer des partenaires existantes \item Rechercher des partenaires par Nom, Adresse ou Ville  \end{itemize}
\medskip
\item \textbf{ Gérer les missions affectées : }
\medskip
\medskip
\begin{itemize} \item Consultation des missions affectées \item Ajout des nouveaux missions affectées \item Modification des missions affectées existantes \item Supprimer des missions affectées existantes \item Rechercher des missions affectées par Code employé, Code partenaire, Code mission ou Date du mission  \end{itemize}
\medskip
\item \textbf{ Gérer les employés : }
\medskip
\medskip
 \begin{itemize} \item Consultation des employés \item Ajout des nouveaux employés \item Modification des employés existantes \item Supprimer des employés existantes \item Rechercher des employés par Code employé \end{itemize}

\end{itemize}


\subsection{Besoins non fonctionnels}

\begin{itemize}
\item \textbf{La convivialité :}
C'est la point la plus important que nous nous concentrons sur pour se différencier d'autres applications et c'est de présenter une interface compréhensible, intuitif, simple et facile à manipuler pour nous permettra d'accroitre la rentabilité et l'efficacité de notre système.
\medskip
\item \textbf{La Disponibilité :}
Le système doit être disponible à tout moment pour l'utilisateur.
\medskip
\item \textbf{La Performance et la capacité :}
Le système doit avoir un temps de réponse et traitement rapide ainsi qu'une capacité de stockage assez grand pour gérer les missions.
\end{itemize}


\subsection{Identification des acteurs}

L'acteur est le rôle assumé par un élément externe qui interagit avec le système étudié. Et pour notre application nous avons un seul acteur qui est le Responsable GRH (Gestion Ressource Humaines). Ce dernier a tous les droits d'accès à l'application. 

\medskip

Il peut gérer les missions, les partenaires, les missions affectées et les employés.
